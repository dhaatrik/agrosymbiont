## 2026-04-22 - Optimize validation state updates
**Learning:** When updating form validation state (e.g., `setErrors` in custom hooks), explicitly check if the new error value matches the existing state value before returning a new object clone. Returning the previous state object directly when values haven't changed prevents unnecessary state thrashing and re-renders during high-frequency events like keystrokes.
**Action:** Always implement an early return in `setState` callbacks if the state conceptually hasn't changed.
## 2026-04-23 - Optimize form inputs with MemoizedInput in InvestorContactForm
**Learning:** In monolithic React forms within this codebase, failing to memoize presentation wrappers like `FormField` results in an O(n) re-render cost for the entire form on every keystroke. Always wrap such repetitive presentation components in `React.memo` and consistently use them for all relevant form inputs rather than mixing them with raw HTML `<input>` elements.
**Action:** When working on form components, always explicitly check if inputs are correctly using `MemoizedInput` or `React.memo` to prevent O(n) re-renders, and refactor any unmemoized `<input>` elements to use the memoized wrapper.
